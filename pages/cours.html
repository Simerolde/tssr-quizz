Switching — Fonctionnement
  • Le switch apprend les adresses MAC en mémorisant le port source de chaque trame → Table CAM
  • Si MAC de destination inconnue → flooding sur tous les ports du VLAN
  • Si MAC de destination connue → unicast vers le port correspondant
  • STP (802.1D) : élit un Root Bridge, bloque les ports redondants pour éviter les boucles
  • RSTP (802.1w) : version rapide de STP, convergence en ~2s vs 30-50s

⭐ À retenir absolument
  • Hub = couche 1, Switch = couche 2 (MAC), Routeur = couche 3 (IP)
  • ARP résout IP → MAC · DNS résout nom → IP
  • Masque /24 = 254 hôtes · /26 = 62 hôtes · /27 = 30 hôtes
  • VLAN natif sur trunk Cisco = VLAN 1 (non tagué par défaut)

Active Directory — Structure
  • Forêt → Domaine(s) → Unités d'Organisation (OU) → Objets (users, groups, machines)
  • GPO : appliqué dans l'ordre LSDOU (Local → Site → Domain → OU). Le dernier appliqué gagne
  • Kerberos : protocole d'authentification AD. Ticket TGT → Ticket de service
  • FSMO : 5 rôles maîtres d'opérations — PDC Emulator, RID Master, Infrastructure Master, Schema Master, Domain Naming Master
  • Outils : dsa.msc (Users & Computers) · gpmc.msc (Group Policy) · dcpromo (promotion DC)
⭐ À retenir absolument
  • /etc/passwd = infos compte (lisible tous) · /etc/shadow = hashs mdp (root uniquement)
  • systemd : enable = démarrage auto · start = lancer maintenant · --now = les deux
  • GPO order : Local → Site → Domaine → OU (le plus spécifique l'emporte)
  • Kerberos : tolérance d'horloge 5 min max (NTP critique en AD)

Chiffrement — Concepts clés
  • Symétrique : même clé pour chiffrer et déchiffrer (AES-256). Rapide. Problème = échange de clé
  • Asymétrique : clé publique (chiffre) + clé privée (déchiffre). RSA, Ed25519. Lent mais sécurise l'échange
  • TLS : combine asymétrique (handshake + échange clé session) + symétrique (données). Best of both worlds
  • Hash : SHA-256, bcrypt. Irréversible. Utilisé pour mots de passe, intégrité
  • PKI : infrastructure de gestion de certificats. CA (Certificate Authority) signe les certificats
⭐ À retenir absolument
  • IDS = passif (détecte + alerte) · IPS = actif (détecte + bloque)
  • HTTP = 80 · HTTPS = 443 · SSH = 22 · RDP = 3389 · DNS = 53 · DHCP = 67/68
  • Principe du moindre privilège : ne donner que les droits strictement nécessaires
  • 2FA = quelque chose qu'on SAIT + quelque chose qu'on POSSÈDE (ou biométrie)

PoE — Power over Ethernet
  • PoE (802.3af) : 15.4W max → téléphones IP, petites caméras
  • PoE+ (802.3at) : 30W max → points d'accès WiFi, caméras PTZ
  • PoE++ (802.3bt) : 60W / 100W → téléphones conférence, mini-PC
  • Le switch PoE doit avoir un budget PoE total suffisant (somme de tous les ports)
  • L'injecteur PoE permet d'alimenter un équipement depuis un switch non-PoE
⭐ À retenir absolument
  • Distance max Ethernet cuivré = 100 mètres (toutes catégories pour 1 Gbps)
  • Câble droit = équipements différents (PC→Switch) · Croisé = équipements identiques (Switch→Switch)
  • Auto-MDIX sur les équipements modernes détecte automatiquement le type de câble
  • VLAN 802.1Q : tag de 4 octets ajouté à la trame Ethernet, ID VLAN sur 12 bits (4094 VLANs)

⭐ Commandes de diagnostic Apache
  • apache2ctl configtest : TOUJOURS vérifier la syntaxe avant reload
  • tail -f /var/log/apache2/error.log : suivre les erreurs en temps réel
  • apache2ctl -S : liste tous les VirtualHosts actifs et leurs bindings
  • 403 Forbidden = problème de droits (chown/chmod ou Require) · 404 = DocumentRoot incorrect · 500 = erreur de config PHP/module
  • En production : utiliser Let's Encrypt (certbot) pour des certificats SSL gratuits et valides

Sauvegarde & Restauration
Stratégies · rsync · tar · robocopy · RAID · RPO/RTO
Stratégie 3-2-1 et types de sauvegardes
TypeCe qui est sauvegardéVitesse backupVitesse restaurationEspace
ComplèteTout à chaque foisLenteRapide (1 seule backup)Important
DifférentielleModifications depuis la dernière COMPLÈTEMoyenne (croît)Moyenne (complète + 1 diff)Moyen
IncrémentaleModifications depuis la dernière sauvegardeRapideLente (complète + toutes incrémentales)Minimal
Règle 3-2-1 : 3 copies · 2 supports différents · 1 hors site
RPO (Recovery Point Objective) = perte de données max acceptable (ex: RPO=4h → backup toutes les 4h)
RTO (Recovery Time Objective) = durée max pour restaurer (ex: RTO=2h → système opérationnel en 2h max)
rsync — Synchronisation Linux
# Synchronisation locale rsync -av /source/ /destination/ # OPTIONS ESSENTIELLES : # -a = archive (récursif + droits + timestamps + liens symboliques) # -v = verbose (affiche les fichiers) # -z = compression pendant le transfert # -P = progression + reprise en cas d'interruption # --delete = supprime les fichiers absents de la source (miroir exact) # --exclude = exclure des fichiers/dossiers # --dry-run (-n) = simulation (TOUJOURS tester avant --delete !) # Sauvegarde vers un serveur distant rsync -avzP --delete /var/www/ user@192.168.1.100:/backup/www/ # Exclure des répertoires rsync -av --exclude='*.tmp' --exclude='.git/' /source/ /destination/ # Sauvegarde vers un serveur distant avec clé SSH rsync -avz -e "ssh -i /root/.ssh/backup_key" /data/ backup@srv-backup:/backup/ # Script de sauvegarde rsync avec log #!/bin/bash rsync -avz --delete /var/www/ /backup/www/ >> /var/log/rsync_backup.log 2>&1 echo "Backup terminé : $(date)" >> /var/log/rsync_backup.log
tar — Archives Linux
# Créer une archive compressée (gzip) tar -czf backup_$(date +%Y%m%d).tar.gz /var/www/html/ # Créer une archive compressée (bzip2 - meilleure compression) tar -cjf backup_$(date +%Y%m%d).tar.bz2 /var/www/html/ # Lister le contenu d'une archive tar -tzf archive.tar.gz # Extraire une archive dans /tmp/ tar -xzf archive.tar.gz -C /tmp/ # Extraire un fichier spécifique tar -xzf archive.tar.gz var/www/html/index.php # OPTIONS : c=créer, x=extraire, t=lister, z=gzip, j=bzip2, v=verbose, f=fichier
Robocopy — Sauvegarde Windows
# Copier un dossier vers un backup (avec sous-dossiers) robocopy C:\Données D:\Backup /S /Z /LOG:C:\logs\backup.log # Miroir exact (attention : supprime les fichiers absents de la source) robocopy C:\Données D:\Backup /MIR /LOG:backup.log # OPTIONS ESSENTIELLES : # /S = sous-répertoires (sans les vides) # /E = sous-répertoires (avec les vides) # /MIR = miroir = /E + /PURGE # /Z = mode redémarrable (reprend si interrompue) # /MT:8 = 8 threads (accélère la copie) # /LOG = fichier de log # /XF = exclure des fichiers (ex: /XF *.tmp *.log) # /XD = exclure des dossiers # Planifier avec le Planificateur de tâches Windows schtasks /create /sc DAILY /tn "Backup Données" /tr "robocopy C:\Données D:\Backup /MIR" /st 02:00
⭐ À retenir absolument
  • rsync --dry-run AVANT --delete : toujours simuler avant de détruire
  • RAID ≠ sauvegarde : le RAID protège contre la panne matérielle, pas contre la suppression accidentelle ou le ransomware
  • Tester la restauration régulièrement : une sauvegarde non testée n'est pas fiable
  • VSS (Shadow Copy) sur Windows permet de sauvegarder les fichiers ouverts
  • sfc /scannow (Windows) : répare les fichiers système corrompus

ITIL — Les processus clés pour le TSSR
Processus ITILDéfinitionExemple concret
Gestion des incidentsRestaurer le service le plus vite possiblePC qui ne démarre plus → dépannage rapide
Gestion des problèmesTrouver et éliminer la cause racine20 PCs crashent → analyse et correctif
Gestion des changementsContrôler les modifications pour éviter les incidentsMise à jour serveur → procédure CAB
Gestion de la configurationMaintenir la CMDB à jourInventaire de tous les assets IT
Gestion des niveaux de serviceDéfinir et respecter les SLARéponse incident P1 en moins de 4h
SLA (Service Level Agreement) = engagement de niveau de service. KPI = indicateurs de performance (taux de résolution, MTTR...). MTTR = Mean Time To Repair.
GLPI — Fonctionnalités essentielles
  • Inventaire : matériel (PC, serveurs, imprimantes), logiciels installés, licences. Automatisé via agent FusionInventory
  • Tickets : incidents et demandes. Workflow : Ouvert → En cours → Résolu → Clôturé. Attribution à un technicien ou groupe
  • CMDB : relations entre assets (ce serveur héberge ces VMs, ce PC utilise cette imprimante)
  • Contrats : maintenance, garanties, fournisseurs avec dates d'expiration
  • Planification : interventions, maintenances préventives
  • Base de connaissances : documentation des solutions pour les tickets récurrents

Wake on LAN (WOL)
# Prérequis : # 1. Activer WOL dans le BIOS/UEFI du PC # 2. Activer sur la carte réseau : # Device Manager → Adaptateurs réseau → Propriétés # → Gestion de l'alimentation → Autoriser le réveil # → Avancé → Wake on Magic Packet → Activé # Envoyer un magic packet depuis Linux # Installation : apt install wakeonlan wakeonlan AA:BB:CC:DD:EE:FF # Depuis PowerShell Windows function Send-WakeOnLan { param([string]$MacAddress) $mac = $MacAddress -replace '[:-]', '' $bytes = [byte[]]@(0xFF,0xFF,0xFF,0xFF,0xFF,0xFF) 0..15 | ForEach-Object { for ($i = 0; $i -lt 6; $i++) { $bytes += [byte]('0x' + $mac.Substring($i*2, 2)) } } $udp = New-Object System.Net.Sockets.UdpClient $udp.Connect([System.Net.IPAddress]::Broadcast, 9) $udp.Send($bytes, $bytes.Length) | Out-Null } Send-WakeOnLan -MacAddress "AA-BB-CC-DD-EE-FF" # Trouver l'adresse MAC d'un PC (depuis Windows) arp -a | findstr "192.168.1.50" # Ou via PowerShell : Get-NetNeighbor -IPAddress 192.168.1.50
⭐ À retenir pour l'examen et les entretiens
  • ITIL : incident = symptôme (restaurer vite) · problème = cause racine (résoudre définitivement)
  • GLPI + FusionInventory = combo standard pour gestion de parc en PME/collectivité
  • WOL nécessite que le PC soit sur le même réseau L2 (broadcast) OU que le routeur gère le "directed broadcast"
  • SLA P1 (critique) : typiquement 4h de RTO · P2 (urgent) : 8h · P3 (normal) : 48-72h
  • Déployer des imprimantes via GPO = zéro intervention utilisateur = bonne pratique

⭐ Règles d'or Cisco IOS
  • ACL standard = près de la destination · ACL étendue = près de la source
  • Toute ACL se termine par un deny ip any any implicite — attention aux blocages involontaires
  • no shutdown est obligatoire sur chaque interface (désactivées par défaut)
  • write memory ou copy run start : TOUJOURS sauvegarder après modification
  • Masque générique (wildcard) = inverse du masque réseau : /24 → 0.0.0.255
